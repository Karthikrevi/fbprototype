✅ Core Features
1. Inventory Table Structure (products)
sql
Copy
Edit
CREATE TABLE products (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  vendor_email TEXT NOT NULL,
  name TEXT NOT NULL,
  description TEXT,
  category TEXT,
  buy_price REAL,
  sale_price REAL,
  quantity INTEGER DEFAULT 0,
  image_url TEXT,
  barcode TEXT UNIQUE
);
2. Batching Table for Inventory Control (FIFO)
sql
Copy
Edit
CREATE TABLE product_batches (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  product_id INTEGER,
  batch_name TEXT,
  quantity INTEGER,
  buy_price REAL,
  arrival_date TEXT,
  FOREIGN KEY(product_id) REFERENCES products(id)
);
💼 Functional Requirements
🔹 Vendor Flow
/erp/products
List of products owned by the vendor (cards or table)

Show:

Name, Image, Quantity

Buy Price (optional)

Sale Price (highlighted)

Stock Summary (from batches)

Edit / View button

/erp/products/add
Add new product (fields: name, desc, category, prices, image upload, barcode, etc.)

Barcode can be manually entered or scanned using a simple JS input

When adding quantity, also create batch entries under product_batches (with batch name and arrival date)

/erp/products/<id>/edit
Editable form:

Change description, prices, barcode

Upload new image

Add or remove batches

Adjust sale price

/erp/products/<id>/view
Product details view page for vendor

Read-only except for "Edit" button

Shows all batch breakdowns, FIFO hint: "Oldest batch will be sold first"

🧾 Extra Notes on Logic
Buy Price: Not shown to users. Used only in backend/profit calculations

Sale Price: Visible to customers

Quantity: Sum of all batch quantities

When a sale happens (simulated later), reduce stock from oldest batch first
