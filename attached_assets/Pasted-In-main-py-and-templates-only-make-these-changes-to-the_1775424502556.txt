PART 1 — New database tables:

Add to init_erp_db() using 
ALTER TABLE IF NOT EXISTS pattern:

CREATE TABLE IF NOT EXISTS payable_entries (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    vendor_id INTEGER,
    payee_name TEXT NOT NULL,
    description TEXT,
    amount REAL NOT NULL,
    amount_paid REAL DEFAULT 0,
    balance_due REAL,
    due_date TEXT,
    category TEXT,
    status TEXT DEFAULT 'unpaid'
        CHECK(status IN ('unpaid','partial','paid')),
    created_at TEXT DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (vendor_id) REFERENCES vendors(id)
)

CREATE TABLE IF NOT EXISTS receivable_entries (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    vendor_id INTEGER,
    payer_name TEXT,
    payer_email TEXT,
    description TEXT,
    amount REAL NOT NULL,
    amount_received REAL DEFAULT 0,
    balance_due REAL,
    due_date TEXT,
    status TEXT DEFAULT 'unpaid'
        CHECK(status IN ('unpaid','partial','paid')),
    created_at TEXT DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (vendor_id) REFERENCES vendors(id)
)

CREATE TABLE IF NOT EXISTS payment_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    vendor_id INTEGER,
    entry_type TEXT CHECK(entry_type IN 
        ('payable','receivable')),
    entry_id INTEGER,
    amount REAL NOT NULL,
    payment_method TEXT NOT NULL
        CHECK(payment_method IN (
        'Cash','Bank Transfer','UPI',
        'Cheque','Card','Credit Note','Other')),
    payment_date TEXT NOT NULL,
    reference_number TEXT,
    notes TEXT,
    created_at TEXT DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (vendor_id) REFERENCES vendors(id)
)

PART 2 — Update accounts_payable route:

Rewrite the accounts_payable route to:

1. Pull from payable_entries table instead
   of expenses table
   
2. On POST — add new payable entry:
   payee_name, description, amount,
   due_date, category
   Set balance_due = amount on creation
   Auto-add to ledger_entries as debit
   to Accounts Payable account

3. Show summary:
   Total outstanding from payable_entries
   where status != paid
   Count of unpaid entries
   Overdue count where due_date < today

4. Show entries table with columns:
   Payee, Description, Amount, Paid, 
   Balance, Due Date, Status
   
5. Each row has a Record Payment button
   that opens a form — no page reload,
   use a modal or inline form — with:
   - Amount being paid number field
   - Payment method dropdown:
     Cash, Bank Transfer, UPI, 
     Cheque, Card, Credit Note, Other
   - Payment date date picker
   - Reference number optional text field
   - Notes optional text field
   - Save Payment button POSTs to
     /erp/finance/payable/pay/<entry_id>

6. /erp/finance/payable/pay/<entry_id> POST:
   Insert into payment_records table
   Update payable_entries amount_paid 
   and balance_due
   If balance_due = 0 set status = paid
   If balance_due > 0 set status = partial
   Add to ledger_entries:
   Debit: Accounts Payable (reducing liability)
   Credit: payment method account (Cash or Bank)

7. Add New Payable button at top that shows
   an inline form or modal with:
   - Payee name text input
   - Description text input
   - Amount number input
   - Due date date picker
   - Category dropdown:
     Supplier, Rent, Utilities, Salaries,
     Equipment, Services, Other
   POSTs to /erp/finance/accounts-payable
