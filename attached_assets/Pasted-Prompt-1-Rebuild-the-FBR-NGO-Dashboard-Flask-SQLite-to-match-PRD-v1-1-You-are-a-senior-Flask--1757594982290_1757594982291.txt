Prompt 1 — Rebuild the FBR NGO Dashboard (Flask + SQLite) to match PRD v1.1

You are a senior Flask engineer working in Replit. Refactor and extend the existing project into a clean, modular FBR NGO Dashboard that implements the features below. Keep it production‑hardened but lightweight for Replit. Use Flask + SQLAlchemy + SQLite, Jinja2 templates, and TailwindCSS (via CDN).

High‑Level Goals
	•	Module: FBRegistry core (UDI + QR + geo + certificates)
	•	App: FBR NGO console (register strays, vaccination with video proof, donation utilization)
	•	App: FDR Vet console (light) for indoor vaccinations, microchip, blood type
	•	Public verify page for QR: /v/<UDI>
	•	Public ledger page (read‑only): donations & utilization

Must‑Have Features (NGO Console)
	1.	Auth & Roles: NGO Admin, NGO Field, Vet (shared), Gov (read‑only for now)
	•	JWT session cookies; simple email/OTP login endpoints
	2.	UDI issuance: Create stray → photos (front/side) → auto UDI with Luhn checksum → generate QR (PNG/SVG)
	3.	Geo‑tagging: Capture lat/lng/accuracy from browser, store with ward_id (text) for now
	4.	Outdoor vaccination: add vaccine event (brand, batch, expiry, dose, route, site, next due) + video proof upload (10–30s, ≤50MB); queue for admin review
	5.	Review queue: NGO Admin approves/rejects vaccination proofs with checklist + comments
	6.	Certificates: On approval, issue PDF e‑certificate with QR that resolves to /v/<UDI> and a JWS signature placeholder
	7.	Donations: View donations to UDI/Drive/NGO; publish utilization with receipts/photos; public ledger updates
	8.	Aggression marker: Set/update (Friendly/Cautious/DoNotApproach)
	9.	Dash widgets: Counters + table views: to‑register, to‑review, due boosters, utilization overdue (D+7, D+14)

Data Model (SQLAlchemy)
	•	Animal(id, udi, short_id, type:Enum(‘p’,‘s’), name, sex, color, breed, approx_age, photos_json, aggression_marker, owner_id, ngo_id, lat, lng, accuracy_m, ward_id, created_at, updated_at)
	•	Vaccination(id, animal_id, type, brand, batch, expiry, dose_ml, route, site, date_time, next_due, mode:Enum(‘outdoor’,‘indoor’), verifier_id, video_url, video_sha256, vet_esign_id, cert_url, lat, lng, accuracy_m, status:Enum(‘pending’,‘approved’,‘rejected’))
	•	Microchip(id, animal_id, chip_id, date, vet_id)

	•	BloodType(id, animal_id, type_dea, verified_on, vet_id)

	•	DonationUtilization(id, donation_id, status:Enum(‘pending’,‘complete’), proof_urls_json, notes, updated_by, updated_at)
	•	User(id, role:Enum(‘ngo_admin’,‘ngo_field’,‘vet’,‘gov’,‘admin’), name, email, phone, password_hash?, otp_secret, ngo_id, last_login)

Key Services to Implement
	•	luhn_check_digit(seq: str) -> str
	•	generate_udi(state='KL', district_code='TVM') -> (udi, short_id)
	•	make_qr_png(payload_url, out_path) using qrcode
	•	sign_jws(payload: dict) -> str (stub with HS256 + env SECRET; make interface to swap to JWS/RS256 later)
	•	generate_certificate_pdf(vaccination_id) -> bytes (ReportLab or WeasyPrint)
	•	hash_video(path) -> sha256

Endpoints (representative)

	•	Vet web: /vet/vaccinations/new?animal_id=, /vet/microchip, /vet/bloodtype
	•	Public: /v/<udi> (verify), /public/ledger
	•	API: /api/animals, /api/animals/<id>/vaccinations, /api/vaccinations/<id>/approve, /api/donations, /api/donations/<id>/utilization

Uploads & Validation
	•	Videos: accept mp4 only, ≤50MB; server‑side size check; compute sha256; store under /uploads/videos/<year>/<month>/.
	•	Photos: compress to ≤1MB each; store under /uploads/photos/.
	•	Anti‑fraud: store device fingerprint header and IP; simple GPS‑mock flag from client; write to audit fields.

UI Requirements (Tailwind)
	•	Dashboard cards: To Review, Due Boosters, Overdue Utilization, New Registrations
	•	Table views with filters and pagination
	•	Forms: progressive disclosure; sticky save; client‑side compression hints for video

Seed & Fixtures

Acceptance Tests (minimum)
	•	Create stray → UDI/QR generated
	•	Submit outdoor vaccination + video → appears in review queue → approve → certificate accessible → /v/<UDI> shows correct status
	•	Donation → ledger entry → utilization update visible on public page

Prompt 2 — Build Government Backend (Data Collection + Read‑Only Console)

Create a separate Flask blueprint gov that exposes APIs + a lightweight console for Kerala DHS users to view/export data collected by NGOs/Vets.

Features
	1.	Gov Auth: email/OTP login for role='gov'
	2.	Dash KPIs: cards + charts (counts only; no PII of pet owners)
	•	Registered animals (pets/strays)
	•	Vaccinated vs unvaccinated; due/overdue boosters
	•	Outdoor proof approval rate
	•	Donations raised + utilization rate
	3.	Geo Views
	•	Map tiles using Leaflet (via CDN); points for strays; ward heatmap (mock ward polygons if needed)
	4.	Exports
	•	CSV/JSON/GeoJSON for: Animals, Vaccinations, Donations, DonationUtilization
	•	Date range, district, NGO filters
	5.	Public Keys
	•	Serve current QR verify public key at /gov/qrkey/current (rotatable via admin)

Endpoints
	•	Web: /gov/login, /gov/dashboard, /gov/map, /gov/exports
	•	API: /api/gov/animals?filters, /api/gov/vaccinations?filters, /api/gov/donations?filters, /api/gov/utilization?filters, /api/gov/export/<entity>.<csv|json|geojson>

Privacy & Redaction
	•	Pet owner PII never shown in gov or public UIs; only aggregate counts
	•	Exact pet home coordinates are not displayed on public/gov maps; show only ward/district aggregates for pets; strays can show points

Config & Env
	•	SECRET_KEY, JWT_SECRET, UPLOAD_DIR, QR_SIGNING_SECRET, MAPBOX_TOKEN? (optional; prefer Leaflet + OSM)

Libraries to add
	•	Flask, Flask_SQLAlchemy, Flask_Migrate, itsdangerous, PyJWT, qrcode, Pillow, reportlab, python-magic, Werkzeug, leaflet (CDN), tailwind (CDN)

Delivery Checklist
	•	Blueprints registered; nav shows NGO / Vet / Gov / Public
	•	Role‑based access enforced on every route
	•	Sample data + screenshots rendered
	•	README with run commands and environment variable examples
