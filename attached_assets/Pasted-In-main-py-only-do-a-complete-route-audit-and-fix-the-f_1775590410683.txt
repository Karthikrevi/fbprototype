PART 4 — Fix back button destinations:

Add a referrer tracking system:
When entering a portal store the
portal name in session:
current_portal = vendor/vet/handler/
                 ngo/admin/pet_parent

In templates the back button should
use this to go to correct dashboard.

In main.py add a helper function:
def get_portal_dashboard():
    if session.get('vendor_email'):
        return '/erp/dashboard'
    elif session.get('vet_email'):
        return '/furrvet/dashboard'
    elif session.get('handler_email'):
        return '/handler/dashboard'
    elif session.get('ngo_email'):
        return '/ngo/dashboard'
    elif session.get('admin_email'):
        return '/admin/dashboard'
    else:
        return '/dashboard'

Make this available as a template
global using:
app.jinja_env.globals['get_portal_dashboard'] 
= get_portal_dashboard
