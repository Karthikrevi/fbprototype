PART 1 — Database:

Add to init_erp_db():

CREATE TABLE IF NOT EXISTS
approval_tokens (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    token TEXT UNIQUE NOT NULL,
    user_type TEXT NOT NULL
        CHECK(user_type IN
        ('furrwings_vet','handler',
        'isolation')),
    user_id INTEGER NOT NULL,
    user_email TEXT NOT NULL,
    is_used BOOLEAN DEFAULT 0,
    is_revoked BOOLEAN DEFAULT 0,
    created_at TEXT
        DEFAULT CURRENT_TIMESTAMP
)

Note: NO expiry column.
Tokens only invalidated by
is_used = 1 or is_revoked = 1.

CREATE TABLE IF NOT EXISTS
verification_checklists (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    application_type TEXT NOT NULL,
    application_id INTEGER NOT NULL,
    checklist_item TEXT NOT NULL,
    item_category TEXT NOT NULL,
    status TEXT DEFAULT 'pending'
        CHECK(status IN
        ('pending','in_progress',
        'passed','failed','waived')),
    notes TEXT,
    verified_by TEXT,
    verified_date TEXT,
    document_url TEXT,
    created_at TEXT
        DEFAULT CURRENT_TIMESTAMP
)

CREATE TABLE IF NOT EXISTS
verification_documents (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    application_type TEXT NOT NULL,
    application_id INTEGER NOT NULL,
    document_name TEXT NOT NULL,
    document_type TEXT NOT NULL,
    file_url TEXT NOT NULL,
    uploaded_by TEXT NOT NULL,
    notes TEXT,
    created_at TEXT
        DEFAULT CURRENT_TIMESTAMP
)

PART 6 — Resend link route:

Add route
/admin/furrwings/vets/<int:vid>/
resend-link
POST:
Admin only

Revoke any existing unused tokens:
UPDATE approval_tokens
SET is_revoked = 1
WHERE user_type = furrwings_vet
AND user_id = vid
AND is_used = 0

Generate new token
Same as approve route but
without changing approval_status

Send email with new link
Flash success with link as backup

Redirect to /admin/furrwings/vets

PART 7 — Password setup route:

Add route /set-password/<token>
GET and POST:
No login required — public route

GET:
Look up token in approval_tokens
If not found:
  Render set_password_invalid.html:
  "Invalid Link"
  "This link is not valid."
  "Contact support if you believe
  this is an error."
  support@furrbutler.com

If is_revoked = 1:
  Render set_password_invalid.html:
  "Link No Longer Valid"
  "This link has been replaced
  with a new one. Please check
  your email for the latest link
  or contact support."

If is_used = 1:
  Check if user is already
  logged in with correct session:
  If yes: redirect to their dashboard
  If no: render set_password_invalid.html:
  "Link Already Used"
  "This link has already been
  used to set a password."
  "Go to login page" button
  linking to correct portal login
  based on user_type

If valid (not used, not revoked):
  Render set_password.html
  Pass: user_type, user_email, token

POST:
  Body: password, confirm_password
  Validate:
    Min 8 characters
    At least 1 number
    At least 1 letter
    Passwords match
  Show inline errors if invalid

  Hash with werkzeug
  Update correct table:
    furrwings_vet → furrwings_vets
    handler → handlers
    isolation → isolation_centers
  Set password = hashed

  Mark token as used:
  UPDATE approval_tokens
  SET is_used = 1
  WHERE token = token

  Auto login:
    Set session keys for user_type
    furrwings_vet:
      furrwings_vet_id
      furrwings_vet_name
      furrwings_vet_email
      furrwings_clinic
    handler:
      handler_id
      handler_name
      handler_email
    isolation:
      isolation_id
      isolation_name
      isolation_email

  Flash: "Welcome! Your password
  has been set successfully."

  Redirect to correct dashboard
