
### WO-0A — Backend Auth & JWT (Flask) — **Backend only**

**Goal:** Implement secure email+password auth with JWT access/refresh tokens, roles, and basic guards. Do **not** touch the React app yet.

**Constraints**

* Keep existing repo structure.
* Do not delete files; add/modify only.
* Use **Flask + SQLAlchemy + Flask-Migrate**; DB = SQLite for now.
* Libs: `flask`, `flask_sqlalchemy`, `flask_migrate`, `flask_bcrypt` or `argon2-cffi`, `flask-jwt-extended`, `python-dotenv`, `email-validator`, `itsdangerous`.
* Use app factory pattern and Blueprints.

**Environment**

* Create/update `infra/.env.example` with:

  ```
  FLASK_ENV=development
  SECRET_KEY=change_me
  JWT_SECRET_KEY=change_me_too
  DATABASE_URL=sqlite:///dev.db
  ACCESS_TOKEN_EXPIRES_MIN=15
  REFRESH_TOKEN_EXPIRES_DAYS=7
  ```
* Load from `.env` in `api/furrbutler/config.py`.

**Models (api/models)**

* `user.py`

  * `id`, `email` (unique, validated), `password_hash`, `role` (enum: pet\_parent, vendor\_groomer, vendor\_boarding, vet, pharmacy, handler, isolation, ngo, gov\_view, admin), `is_email_verified` (bool), `created_at`, `updated_at`.
  * Methods: `set_password(plain)`, `check_password(plain)`.
* `token_blocklist.py`

  * `id`, `jti` (unique), `created_at` (for refresh/access revocation).
* Ensure `__all__` imports in `api/models/__init__.py`.

**Extensions (api/extensions.py)**

* Initialize: `db`, `migrate`, `bcrypt` (or `argon2`), `jwt`, `cors` (if needed).

**RBAC (api/rbac.py)**

* Helper `require_roles(*roles)` that checks `get_jwt()` claims for `role`. Return 403 JSON on fail.

**Blueprint: Auth (api/blueprints/auth)**

* `POST /api/auth/register`
  Body: `{email, password, role}` (role defaults `pet_parent`, restrict high-priv roles to admin in future).

  * Validate email format + unique.
  * Hash password.
  * Create user with `is_email_verified=False` (we’ll add verification later).
* `POST /api/auth/login`
  Body: `{email, password}`

  * Verify user & password.
  * Return `{access_token, refresh_token, user: {id,email,role,is_email_verified}}`
  * Access token: 15 min; Refresh: 7 days (from env).
* `POST /api/auth/refresh`

  * Requires refresh token.
  * Issues new access token.
* `POST /api/auth/logout`

  * Accepts either access or refresh token; add `jti` to `token_blocklist`.
* `GET /api/auth/me`

  * Requires access token.
  * Returns current user profile.
* Error format: JSON `{error: {code, message}}`.

**JWT config**

* Set additional claims on token creation: `{"sub": user.id, "role": user.role, "email": user.email}`.
* `@jwt.token_in_blocklist_loader` checks `token_blocklist`.
* Standardize auth errors with JSON messages.
* Rate-limit optional (skip for now).

**Migrations & Seeds**

* Create migrations for new models.
* Add `api/seeds/seed_admin.py`: creates one admin user if not exists (email: `admin@example.com`, password: `Admin@123`, role: `admin`).
* Add `infra/scripts/manage.py` with commands: `init-db`, `upgrade-db`, `seed-admin`.

**Tests (api/tests/test\_auth.py)**

* Register → Login → Me → Refresh → Logout (blocklisted) → Access denied on old token.
* Bad password and duplicate email cases.

**CORS**

* If needed for local dev, enable CORS for `/api/*` with strict origins.

**Documentation**

* Update `docs/API_GUIDE.md` with the 5 endpoints above, request/response JSON examples.
* Add OpenAPI stub in `common/openapi/auth.yaml` (paths, schemas).

**Acceptance Criteria**

* `POST /api/auth/register` creates user; duplicate email rejected.
* `POST /api/auth/login` returns valid access & refresh tokens and user payload.
* `GET /api/auth/me` works with access token; 401 without/invalid.
* `POST /api/auth/refresh` issues new access token using refresh token.
* `POST /api/auth/logout` blocklists token; subsequent use of that token returns 401.
* All responses are JSON; no HTML templates.
* `pytest -q` passes for `api/tests/test_auth.py`.
* `flask db upgrade` succeeds; seed admin script creates admin user.
* No changes to the React app yet.

**Do not proceed** to frontend wiring in this task. Finish backend auth cleanly first.
